🎯 專案背景與挑戰

專案的複雜性

基金資訊觀測站是台灣金融業的核心基礎設施，服務超過1,000檔基金的資訊揭露與申報需求。2.0專案的複雜度體現在三個維度：系統整合、階段交付與多方協調。

維度一：6大平台整合架構

業務需求
需將舊有獨立運作的多個子系統整合為統一生態系統，建立標準化的前後台架構。

6大平台架構

主平台 (www.fundclear.com.tw)

統一入口、基金搜尋、ESG基金、數據統計、投資教育

期信公告平台

法規申報功能

設計挑戰
  • • 如何在不中斷現有服務的情況下，逐步整合6大平台？

  • • 如何確保資料同步與系統穩定性？

維度二：三階段交付策略

專案時程規劃

採用敏捷開發方法，將兩年的專案拆分為三個獨立階段，每階段經過功能開發 → 集保測試 → 資安檢測 → 驗收上線的完整流程。

  • • 關鍵里程碑：8/4首批交付 → 10/31資安完成 → 11/30驗收上線

  • • 關鍵里程碑：9/20測試完成 → 10/31功能上線
第三階段（2024.08 - 2025.06）
  • • 期信申報平台翻新、期信公告平台
  • • 境外公告平台、境內公告平台
  • • 關鍵里程碑：5/28整合測試 → 6/30最終上線

專案經理面臨的核心挑戰

挑戰 1：複雜的時程管理

情境
90個功能模組需在三階段中精準交付，每階段涉及開發、測試、資安檢測等多個環節，任何一個環節延遲都會影響整體時程。

我的應對方式

建立每週更新的Azure DevOps Dashboard，追蹤所有功能的開發狀態（To Do / In Progress / In Review / QA / Done）。每月召開專案會議，向集保報告進度與風險，確保所有利害關係人對齊。

挑戰 2：跨部門溝通與決策

向上溝通
協調集保4大部門的需求，確保業務需求與技術實現的可行性
向下溝通
管理25人開發團隊，協調6大技術組別的分工與整合
橫向溝通
與資安檢測單位協調測試時程，確保符合金融業資安規範

挑戰 3：品質控管與測試

金融業對系統穩定性與資安的要求極高，每個階段都需經過完整的測試流程。

  • 功能測試：確保所有功能符合需求規格
  • 集保測試：客戶方進行UAT測試，提交Bug單
  • 資安檢測：第三方資安單位進行漏洞掃描與滲透測試
  • 整合測試：確保6大平台整合後的系統穩定性

第二步：完整的測試與品質管理

以風險管理為核心的測試策略

實際案例：Bug 追蹤管理

第一階段交付後，集保測試階段提交了多筆Bug單。我建立了完整的Bug追蹤機制：

  • • 每個Bug都標記優先級（Critical / High / Medium / Low）
  • • Critical與High優先級的Bug必須在下個Sprint前修正
  • • 所有Bug修正後需再次回歸測試，確保沒有引入新問題
  • • 建立Bug修正記錄，作為後續階段的參考

撰寫測試腳本與測試計畫

針對每個功能模組撰寫詳細的測試腳本，定義測試情境與驗收標準。

測試計畫結構

  • • 前置條件：測試前需準備的資料與環境
  • • 測試步驟：詳細的操作流程
  • • 預期結果：每個步驟應該看到的結果
  • • 異常情境：錯誤處理與邊界條件測試
  • • 跨平台驗證：不同瀏覽器與裝置的測試

第三步：主動發現問題並提出解決方案

實例：UI/UX 配色調整需求

原始需求：設計師提供的視覺稿需要調整配色以符合集保的企業識別。

我發現的問題：

  • • 6大功能模組（首頁、基金搜尋、ESG基金、數據統計、投資教育、公告資訊）各自的配色需求沒有明確定義
  • • 缺少完整的頁面盤點清單，無法確保所有頁面都符合UI/UX標準

我提出的解決方案：

建立完整的頁面盤點清單，逐頁確認UI/UX配色調整需求；與設計師和集保確認每個模組的主色調、輔助色、文字顏色等細節；建立設計規範文件，確保所有頁面風格一致。

成果：確保所有頁面符合集保的企業識別，提升整體視覺一致性。

🎓 核心學習與反思

從這個專案學到什麼？

1. 階段性交付的重要性

透過三階段交付，能讓利害關係人提早看到成果，及時調整需求，降低專案風險。每個階段都是一個完整的「開發→測試→資安→上線」循環，確保品質。

2. 溝通透明化是關鍵

每週更新Dashboard與每月進度會議，讓所有人掌握專案狀態，減少誤解與重工。透明的溝通建立信任，讓團隊更有向心力。

3. 測試腳本的價值

事先定義測試腳本與驗收標準，能大幅縮短測試時間，確保雙方對交付品質的共識。測試不是開發完成後的附加工作，而是專案規劃時就應納入的核心環節。

4. 跨部門協作的挑戰

金融業專案涉及多個部門（業務、IT、資安、法規），PM需具備高度的溝通與協調能力，理解各部門的需求與限制，找到平衡點。

給自己的話

專案管理不只是時程管理，更是人的管理。成功的專案來自於清晰的溝通、透明的進度、主動的問題發現，以及對品質的堅持。
